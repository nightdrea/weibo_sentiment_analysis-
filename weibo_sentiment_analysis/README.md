# 微博舆情分析系统

## 项目简介

微博舆情分析系统是一个基于 Python 技术栈的数据分析平台，用于爬取、分析和可视化微博数据，帮助用户了解微博上的舆情动态。

## 技术路线

### 1. 后端技术
- **Web 框架**: Flask - 轻量级 Python Web 框架，用于构建 RESTful API 和 Web 页面
- **数据库**: SQLite - 轻量级关系型数据库，用于存储用户数据、微博帖子和评论
- **数据爬取**: MediaCrawler - 强大的社交媒体爬取工具，支持微博数据的高效爬取
- **认证系统**: Flask-Login - 提供用户登录、注册和会话管理
- **安全措施**: Flask-WTF (CSRF 保护)、Flask-Limiter (速率限制)、密码哈希
- **性能优化**: 数据库索引、分页查询、缓存机制

### 2. 前端技术
- **HTML/CSS**: 构建响应式用户界面
- **JavaScript**: 实现交互功能
- **ECharts**: 数据可视化库，用于生成各种图表
- **ECharts-Wordcloud**: 词云图生成库

### 3. 数据处理
- **数据爬取**: 使用 MediaCrawler 爬取微博热搜、帖子和评论
- **数据存储**: 使用 SQLite 数据库存储结构化数据
- **数据可视化**: 使用 ECharts 生成各种图表，包括折线图、饼图、词云图等
- **定时任务**: 使用 schedule 库实现自动定期爬取功能

## 项目实际意义

1. **舆情监测**: 实时监测微博上的热门话题和舆情动态，帮助企业、政府和个人了解公众关注点

2. **数据分析**: 对微博数据进行深度分析，包括情感分析、用户行为分析、内容趋势分析等

3. **决策支持**: 基于数据分析结果，为企业营销、品牌管理、危机公关等提供决策支持

4. **研究价值**: 为社会学、传播学等领域的研究提供数据支持和分析工具

5. **技术演示**: 展示如何构建一个完整的 Web 应用，包括前端、后端、数据库、爬虫等多个技术领域的集成

## 功能特性

- **用户管理**: 注册、登录、退出功能
- **数据爬取**: 支持关键词搜索爬取和热搜话题爬取
- **情感分析**: 分析微博内容的情感倾向
- **数据可视化**: 生成多种图表展示数据
- **词云分析**: 生成微博内容、评论和用户的词云图
- **IP 分析**: 分析微博用户的 IP 分布
- **搜索功能**: 支持搜索微博内容和评论
- **数据导出**: 支持导出数据为 CSV 和 JSON 格式
- **定时爬取**: 自动定期爬取微博数据

## 如何使用

### 1. 环境准备

1. 确保已安装 Python 3.7+
2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 安装 MediaCrawler 依赖：
   ```bash
   # 进入 MediaCrawler 目录
   cd MediaCrawler
   pip install -r requirements.txt
   ```

### 2. 启动系统

1. 进入项目目录：
   ```bash
   cd weibo_sentiment_analysis
   ```

2. 启动应用：
   ```bash
   python run.py
   ```

3. 访问系统：
   - 打开浏览器，访问 http://localhost:5000
   - 首次使用需要注册账号

### 3. 主要功能使用

#### 数据爬取
1. 点击左侧菜单中的 "数据爬取" 选项
2. 输入关键词，选择爬取页数和是否爬取评论
3. 点击 "开始爬取" 按钮
4. 等待爬取完成，查看爬取结果

#### 数据分析
1. 点击左侧菜单中的各个分析选项，如 "舆情分析"、"数据可视化"、"微博文章分析" 等
2. 查看系统生成的各种图表和分析结果

#### 词云分析
1. 点击左侧菜单中的 "词云图" 相关选项
2. 查看系统生成的词云图，了解热门词汇

#### 数据导出
1. 点击左侧菜单中的 "数据导出" 选项
2. 选择数据类型（微博帖子或评论）和导出格式（CSV 或 JSON）
3. 点击 "导出" 按钮，下载数据文件

#### 搜索功能
1. 点击左侧菜单中的 "搜索功能" 选项
2. 输入搜索关键词，选择搜索类型（全部、微博帖子或评论）
3. 点击 "搜索" 按钮，查看搜索结果

## 系统配置

### 数据库配置
- 默认使用 SQLite 数据库，存储在 `instance/weibo_sentiment.db`
- 如需改为 MySQL 数据库，修改 `app/__init__.py` 中的数据库连接配置

### 定时爬取配置
- 在 `app/tasks.py` 中可以调整定时爬取的频率和爬取内容

### 安全配置
- 在 `app/__init__.py` 中可以修改 SECRET_KEY 为更复杂的密钥

## 注意事项

1. 系统使用 MediaCrawler 进行数据爬取，需要确保 MediaCrawler 能够正常运行
2. 爬取数据时，请注意遵守相关法律法规和平台规则
3. 系统默认使用开发服务器，生产环境部署时建议使用 WSGI 服务器
4. 定时爬取功能会在系统启动时自动启动，每小时执行一次

## 故障排除

1. **无法访问系统**：检查系统是否已启动，端口是否被占用
2. **爬取失败**：检查 MediaCrawler 是否正确安装，网络连接是否正常
3. **数据显示异常**：检查数据库连接是否正常，数据是否正确存储
4. **系统运行缓慢**：考虑优化数据库查询，增加缓存机制
